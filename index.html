# XMiles Dashboard Dashboard de producao e saldos de milhas, alimentado pela planilha de controle. ## Setup no GitHub 1. Criar repositorio no GitHub (ex: `xmiles-dashboard`) 2. Clonar e copiar os arquivos: ``` git clone https://github.com/SEU_USER/xmiles-dashboard.git cd xmiles-dashboard cp /caminho/dashboard-xmiles/* . ``` 3. Push inicial: ``` git add . git commit -m "Dashboard inicial" git push ``` 4. Ativar GitHub Pages: - Settings > Pages > Source: Deploy from branch > main > / (root) > Save ## Como atualizar Sempre que atualizar a planilha: ```bash cd xmiles-dashboard python3 extract_data.py "../CONTROLE CONTAS LIVELO.xlsx" git add data.json git commit -m "Atualizar dados $(date +%Y-%m-%d)" git push ``` O dashboard no GitHub Pages sera atualizado automaticamente em ~1 minuto. ## Arquivos - `index.html` - Dashboard (abre no navegador) - `extract_data.py` - Script que le a planilha e gera data.json - `data.json` - Dados resumidos (usado pelo dashboard) - `data_full.json` - Dados completos com cada conta (backup) ## Automacao (opcional) Para atualizar automaticamente, crie um GitHub Action: ```yaml # .github/workflows/update.yml name: Update Dashboard on: push: paths: ['CONTROLE CONTAS LIVELO.xlsx'] jobs: update: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-python@v5 with: { python-version: '3.11' } - run: pip install pandas openpyxl - run: python extract_data.py - run: | git config user.name "GitHub Actions" git config user.email "actions@github.com" git add data.json git diff --staged --quiet || git commit -m "Auto-update $(date +%Y-%m-%d)" git push ```
